## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/moncash_flutter_plus.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes
6. Commit your changes: `git commit -m "Add your message"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Development Setup

### Prerequisites
- Flutter SDK 3.12 or higher
- Dart SDK 2.15.1 or higher

### Running the Example App
```bash
cd example
flutter pub get
flutter run
```

## Code Style

- Follow [Effective Dart](https://dart.dev/guides/language/effective-dart) guidelines
- Use `flutter analyze` to check for issues
- Add dartdoc comments for public APIs
- Keep code formatting consistent with `dart format`

## Testing

- Add unit tests for new features
- Ensure all tests pass: `flutter test`
- Aim for >80% code coverage

## Pull Request Guidelines

- **One feature per PR**: Keep pull requests focused on a single feature or fix
- **Write clear commit messages**: Explain what and why, not how
- **Update documentation**: Update README.md if you add new features
- **Add tests**: Include tests for new functionality
- **Update CHANGELOG.md**: Add an entry describing your changes

## Reporting Issues

When reporting issues, please include:
- Flutter version (`flutter --version`)
- Device/emulator information
- Steps to reproduce
- Expected behavior
- Actual behavior
- Error messages and stack traces

## Feature Requests

We welcome feature requests! Please:
- Check existing issues first to avoid duplicates
- Clearly describe the feature and its use case
- Explain why it would be useful to other users

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Help create a welcoming environment for all contributors

## Questions?

Feel free to open an issue with the `question` label or reach out to the maintainers.
